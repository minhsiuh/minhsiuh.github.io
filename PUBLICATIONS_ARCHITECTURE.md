# Publications Architecture (Canonical)

Last updated: 2026-03-10

## 1) Data Source of Truth

`publications.html` is rendered from JSON files in `data/`:

- `data/publications.arxiv.json`
- `data/publications.books.json`
- `data/publications.journals.json`
- `data/publications.conferences.json`
- `data/publications.talks.json`

Each file uses:

```json
{
  "section": "journals",
  "updatedAt": "...",
  "items": [
    {
      "id": "journals-1",
      "contentHtml": "Authors. Title. Venue... [links]"
    }
  ]
}
```

---

## 2) Rendering Pipeline in `publications.html`

The page loads JSON via `fetch()` and renders by mode:

- arXiv: `mode='card'`
- books: `mode='book-card'`
- journals: `mode='journal-card'`
- conferences: `mode='conference-card'`
- talks: `mode='talk-card'`

Main function:

- `renderListFromJson(listId, dataFile, mode)`

### Shared parsing helpers

- `parseItem(contentHtml)`
- `parseJournalHead(head)`
- `normalizeAuthors()`
- `cleanTitle()`

Goal: split into stable fields:

- title
- authors
- reference/venue row
- optional links/actions row

---

## 3) Section UI Contract

### arXiv

Rows:
1. title
2. authors
3. `arXiv:xxxx.xxxxx (year)`
4. actions: copy plain / copy BibTeX

### books / journals / conferences

Rows:
1. title
2. authors
3. venue/ref info
4. actions row:
   - `[arXiv] [DOI] | Copy citation (plain) Copy Bibtex`

### referred talks (without proceedings)

Rows:
1. title
2. authors
3. conference info

No fourth row in talks.

---

## 4) Copy Action Rules

### Copy citation (plain)

- Must copy citation body text only.
- Must NOT include bracket link tokens like `[arXiv:...]` or `[DOI]`.

### Copy Bibtex

- Generated by typed builder `buildBibtex(...)`.
- Uses encoded `data-copy-text` payload on each button.
- Clipboard path uses robust fallback when direct clipboard API is unavailable.

---

## 5) BibTeX Type Policy

- arXiv -> `@misc`
- journal -> `@article`
- conference -> `@inproceedings`
- book/proceedings -> `@book` (with special-case `@incollection` override when explicitly defined)

### Mandatory rule

Only include fields that are explicitly known.
Do **not** invent missing values.

### Current curated hard overrides

- TQC 2024 LIPIcs DOI `10.4230/LIPIcs.TQC.2024.8` -> full structured inproceedings fields.
- Books-3 (`BrunHsieh2013`) -> exact `@incollection` canonical BibTeX.

---

## 6) Name/Style Normalization Decisions

Standardized long journal names where requested:

- `npj Quantum Inf` -> `npj Quantum Information`
- `Quantum Mach. Intell.` -> `Quantum Machine Intelligence`
- `Phys. Rev. Lett.` -> `Physical Review Letters`
- `J. Chem. Inf. Model.` -> `Journal of Chemical Information and Modeling`

Title cleanup:

- trailing period in parsed title is removed.

Author cleanup:

- normalize `and` separators
- remove accidental double commas

---

## 7) Validation / QA Checklist (operational)

For each edited item, verify:

1. Title row is title only (no venue contamination)
2. Authors row only contains authors
3. Venue row contains conference/journal/book context
4. Copy plain excludes link tokens
5. Copy BibTeX returns expected type and required known fields
6. No fabricated number/volume/pages if unknown

For batch changes, run validator:

```bash
node scripts/validate-publications.mjs
```

Optional link-check mode:

```bash
node scripts/validate-publications.mjs --check-links
```

---

## 8) Change Management Rules

- Small reversible diffs
- Section-by-section rollout (avoid global risky refactor in one shot)
- Commit frequently with clear scope
- If parsing uncertainty exists, collect and confirm with user before mass edits
